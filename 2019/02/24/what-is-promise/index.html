了解JavaScript中的Promise

Gokul NK
2018年7月16日
原文链接

我先承诺，在这篇文章的最后，你会更好地了解JavaScript Promises。

我与JavaScript有一种“爱与恨”的关系。但是，JavaScript总是对我很有吸引力。在过去10年中，我一直致力于Java和PHP，但JavaScript看起来非常不同，但很有趣。我没有花足够的时间在JavaScript上并且一直在努力弥补它。

Promises是我遇到的第一个有趣话题。我一次又一次地听到有人说Promises会让你免于回调地狱。虽然这可能是一个令人愉快的副作用，但Promise还有更多，这是我迄今为止能够弄清楚的。

背景

当你第一次开始使用JavaScript时，它会有点令人沮丧。你会听到有些人说JavaScript是同步编程语言，而其他人则认为它是异步的。您会听到阻塞代码，非阻塞代码，事件驱动设计模式，事件生命周期，函数堆栈，事件队列，冒泡，polyfill，babel，angular，reactJS，vue JS以及大量其他工具和库。不要烦恼。你不是第一个。这也有一个术语。它被称为JavaScript疲劳。这条推文很好地捕获了它。

如果您想了解有关JavaScript疲劳的更多详细信息，请查看以下文章。有一个原因，这篇文章在Hackernoon上获得了42k的鼓掌:) 2016年学习javascript的感觉

JavaScript是一种同步编程语言。但是由于回调函数，我们可以使它像异步编程语言一样运行。

外行的Promise

JavaScript中的Promise与您在现实生活中所做的Promise非常相似。首先让我们看看现实生活中的Promise。

字典中的Promise定义如下

Promise ：名词：保证一个人会做某事或某个事情会发生。

那么当有人向你许诺时会发生什么？

Promise可以保证您可以完成某些事情。他们（谁做出Promise）是自己做的，还是由他人完成，这是无关紧要的。他们为您提供保证，您可以根据这些保证计划某些事情。
Promise可以保留或破坏。
当一个Promise被保留时，你会期待一些超出Promise的东西。您可以利用Promise的输出来进行进一步的行动或计划。
当一个Promise被打破时，你想知道为什么做出Promise的人无法跟上他的交易。一旦您知道原因并确认Promise已被破坏，您就可以计划下一步该做什么或如何处理它。
在做出Promise时，我们所拥有的只是一种保证。我们将无法立即采取行动。我们可以决定并制定在保留Promise（因此我们预期结果）或破坏时需要做什么（我们知道原因，因此我们可以计划应急）。
有可能你根本没有回复那个做出Promise的人。在这种情况下，您更愿意保持时间阈值。如果做出Promise的人在10天内没有回复我，我会认为他有一些问题而且不会履行诺言。所以，即使这个人在15天后回到你身边，你也不再重要了，因为你已经制定了替代计划。

JavaScript中的Promise

根据经验，对于JavaScript，我总是阅读MDN Web Docs的文档。在所有资源中，我认为它们提供了最简洁的细节。我从MDSN Web Docs中读取了Promises页面，并使用代码来解决它。

理解Promise有两个部分。Promise的制定和Promise的处理。虽然我们的大多数代码通常都会满足其他图书馆创建的Promise的处理，但完整的理解将对我们有所帮助。一旦你跨过初学者阶段，理解“创造Promise”同样重要。

创造Promise

让我们看看创建新Promise的签名。

构造函数接受一个名为executor的函数。此executor函数接受两个参数resolve，reject然后依次为函数。Promise通常用于更容易处理异步操作或阻塞代码，其示例包括文件操作，API调用，DB调用，IO调用等。这些异步操作的启动发生在executor函数内。如果异步操作成功，则resolve通过promise的创建者调用该函数来返回预期结果。同样，如果出现意外错误，则通过调用reject函数传递原因。

在结束之前，我想列出我遵循的所有拇指规则，让我的头脑保持在承诺周围。

Thumb使用promises的规则

  1. 无论何时使用异步或阻塞代码，都要使用promises。
  2. resolve映射到then并reject映射到catch所有实际目的。
  3. 确保为所有承诺编写两者 .catch和 .then方法。
  4. 如果在两种情况下都需要做某些事情 .finally
  5. 我们只有一次改变每个承诺。
  6. 我们可以为单个承诺添加多个处理程序。
  7. Promise对象中所有方法的返回类型，无论是静态方法还是原型方法，都是aPromise
  8. 在Promise.all承诺的顺序被保持在值变量而不管其中承诺首先解决。
