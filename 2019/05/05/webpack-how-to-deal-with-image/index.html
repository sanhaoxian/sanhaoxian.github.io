How does webpack handle the image reference path in the project?

发表于 2019-05-05 | 分类于 Excerpt&Note | 浏览

webpack中是如何处理图片路径的呢？

主要是两种处理方式：

  • 模块化
  • 非模块化

通过js引入的方式，或者在js中引入我认为就是模块化，它会被webpack识别并认为它是一个模块；
那么它会通过url-loader和file-loader两个loader去处理；
这里通过引用两篇学习文章(需要科学上网才能够浏览)：
Handling Images；
File-loader for large size images;

如果是通过模板文件引入的话，类似于直接在html、或则vue的template、.pug等之类的文件中直接在“”标签内嵌“background-image”或者是css中“background-image”中引入图片，那么webpack是不会对这些路径进行处理的，解决方法我觉得可以把写法写成js引入来解决，eg:${require(‘../../static/xxx.png’)}; 或者是通过webpack的插件来解决；
同样是引入优秀文章进行学习（翻译很累，虽然我只是用谷歌翻译）
Copy all images/files to a folder using copy-webpack-plugin
