webpack中extractTextPlugin插件问题

发表于 2018-12-18 | 分类于 Record | 浏览

extract-text-webpack-plugin插件
webpack-extractTextPlugin相关概念，传送门

这个插件的作用大概就是：它会将所有的入口 chunk(entry chunks)中引用的 *.css，移动到独立分离的 CSS 文件。因此，你的样式将不再内嵌到 JS bundle 中，而是会放到一个单独的 CSS 文件（即 styles.css）当中。简单一点说就是用来分离css的。

问题描述： 在下午使用vue-cli打包项目构成中，遇到bootstrap的字体文件无法显示，同时通过后台查看发现，字体文件的请求路径发生错误，访问的是不存在的地址；

原因(个人理解，不知道准不准确)：在利用webpack构建工具的时候，css默认是在css处在的static目录作为根目录，而当在.css文件中引入font字体文件的时候默认是以css下为根目录去请求，所以后台会出现：
/static/css/static/font/xxxxxx.xxx

解决方法：在相应的webpack配置中，找到css相关的配置。找到extractTextPlugin;

设置publicPath: ‘../../‘后，字体文件能够正确请求路径了！

慢慢搜索，弄懂这个插件究竟是干什么的，相应的就要学习很多大牛刨祖坟式的去思考问题：
问题一: webpack是处理打包css的？
问题二：webpack打包css方式有什么不同？

webpack本身不对css进行处理，而是利用loader对css或者相应的css预处理的sass等进行处理；

一下就是webpack处理css方式，一般是结合使用：
style-loader：将模块的导出作为样式添加到 DOM 中
css-loader： 解析 CSS 文件后，使用 import 加载，并且返回 CSS 代码

那么webpack就要通过相应的loader去处理css啦，常用的有style-loader、css-loader；通过配置不同的loader处理不同的css文件，在这里需要注意的一点是：使用loader是需要注意书写顺序的，webpack默认是从右到左逐个loader使用，所以处理顺序有先后之分，loader不能乱放，例如使用sass,那么使用就是：

那么我个人立即webpack对css的处理是：显示利用相关插件对css进行提取、分离什么的，然后通过相应的loader规则将css资源转换成JavaScript模块，然后嵌入模板文件中的style中，最终生效渲染；
可以使用提取css的及合成一个css文件插件，当然也可以不使用，依照项目需求吧；

除此之外还有一个purifycss-webpack插件，用于消除冗余的css代码，结合上述的插件进行处理
