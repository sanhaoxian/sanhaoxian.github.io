SPA初步了解

SPA(single page web application)单页web应用,这样设计的主要意义:

  • 前后端的分离
  • 减轻服务器的压力
  • 增强用户体验
  • Prerender预渲染优化SEO

前端路由的两种主流方法的实现原理:1、History API;2、Hash；

H5新增的History API

重点是history.pushState()和history.reoplaceState()这两个API

两个API都接收三个参数,分别是：

  • 状态对象（state object）：一个javascript对象，与用pushState()方法创建的心里是记录条目关联。无论何时用户导航道新创建的状态，popstate事件都会被触发，并且事件对象的state属性都包含历史记录条目的状态对象的拷贝。
  • 标题（title） firefox浏览器目前会忽略该参数，考虑道未来可能会对该方法进行修改，传入一个空字符串比较安全，或者可以传入一个简短的标题，表明将要进入的状态。
  • 地址（URL） 新的历史记录条目的地址，浏览器不会在调用pushState()方法后加载该地址，但之后，可能会试图加载，例如用户重启浏览器，新的URL不一定是绝对路径；如果是相对路由，他将以当前的URL为基准，传入的URL与当前的URL是同源的，否则pushState()会抛出异常。该参数的可选的；

相同之处是都会操作浏览器的历史记录,而不会引起浏览器页面的刷新；不同之处在于：pushState会增加一条新的历史记录，而replaceState则会替换但前的历史记录

URL的Hash

在h5还没有流行开来之前，spa一般都采用URL的hash()作为锚点，获取到#后的值，并监听其改变，在进行渲染对应的子页面，

eg: 地址为：http://localhost:8888/#/abc,那么利用localtion.hash输出内容就为#/abc。
